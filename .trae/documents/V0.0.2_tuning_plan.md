# V0.0.2 调优计划

## 概要
对 V0.0.2 进行两项交互优化：
1. 审判页面：取消自动切换，改为纯手动按钮切换
2. 答题页面：增加"回到上一题"功能，支持修改已选答案

## 当前状态分析

### TrialPage（`src/pages/TrialPage.tsx`）
- 有 5 秒自动轮播的 `useEffect`（L18-L31）
- `isAutoPlaying` 状态始终为 `true`，无用户关闭入口
- 已有 `handlePrev` / `handleNext` 手动切换函数
- 自动播放和手动点击会互相干扰（手动点击后定时器重置）

### QuizPage（`src/pages/QuizPage.tsx`）
- 答案追加式存储：`[...answers, key]`，无法修改历史答案
- 选中选项后 380ms 自动跳转下一题，无返回机制
- 进度条区域（L132-L145）仅有视觉指示，无交互按钮

---

## 改动一：TrialPage 取消自动切换

### 文件：`src/pages/TrialPage.tsx`

**删除自动播放相关代码：**
- 删除 `isAutoPlaying` state（L13）
- 删除 `autoPlayRef`（L14）
- 删除整个 `useEffect` 自动播放逻辑（L18-L31）
- 删除 `useEffect` 和 `useRef` 的 import（如果不再需要）

**保留：**
- `handlePrev` / `handleNext` 函数不变
- 底部"上一位"/"下一位"按钮不变
- 进度指示器不变
- 所有 Framer Motion 动画不变

---

## 改动二：QuizPage 增加"回到上一题"

### 文件：`src/pages/QuizPage.tsx`

**新增 `handleGoBack` 函数：**
- 检查 `currentIndex > 0` 且 `!isAnimating`
- 设置 `direction = -1`（向右滑入上一题的动画）
- 回退后恢复 `selectedKey` 为 `answers[currentIndex - 1]`（回显之前的选择，若未答过则为 null）
- 设置 `isAnimating = false`

**修改 `handleSelect` 函数：**
- 判断 `currentIndex < answers.length`（是否为修改已有答案）
- 如果是修改：`newAnswers[currentIndex] = key`（替换该位置的答案）
- 如果是新答：保持原有 `[...answers, key]` 追加逻辑不变

**修改底部进度条区域（L132-L145）：**
- 外层 div 改为 `relative` 布局
- 新增 `←` 箭头按钮（仅图标，无文字）：
  - 绝对定位 `absolute left-0 top-1/2 -translate-y-1/2`
  - 尺寸：`w-8 h-8`，图标 20px
  - 颜色：默认 `text-zinc-500`，hover `text-yellow-400`
  - 条件渲染：`currentIndex > 0` 时显示
- 进度条保持 `flex justify-center` 居中不变

**回退时的视觉回显：**
- 回退到已答过的题目时，`selectedKey` 自动恢复为之前的选择
- 用户可以看到之前选了什么，也可以重新选择别的选项

## 验证步骤

1. `npm run build` 编译通过
2. `npm run dev` 启动开发服务器
3. 审判页面验证：
   - 页面加载后不会自动切换审判员
   - 点击"上一位"/"下一位"按钮正常切换
   - 切换动画流畅（左滑/右滑）
   - 最后一位时按钮显示"查看结果"
4. 答题页面验证：
   - 第一题时箭头按钮不显示
   - 答完第 2 题后进度条左侧出现 `←` 箭头
   - 点击回退，能看到之前选的选项高亮显示
   - 重新选择不同选项，答案正确更新
   - 继续前进，最终结果与修改后的答案一致
   - 最后一题选择后正常跳转到审判页面
