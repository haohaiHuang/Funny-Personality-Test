## 1. 项目背景

### 1.1 产品定位

**职场毁型人格测试 + 人物审判庭** 是一款融合趣味心理测试与批判性反思的互动产品。用户在完成搞笑风格的"职场毁型人格"测试后，将接受来自历史/当代人物组成的"审判庭"对其进行多角度深度剖析，最终通过幽默讽刺与哲学思辨的结合，实现**"搞笑吸引 → 批判性反思"**的产品定位。

### 1.2 业务目标

- **传播目标**：打造具有话题性和传播力的职场内容IP，通过趣味测试吸引用户参与
- **用户目标**：让用户在娱乐中完成自我认知的反思，发现职场行为模式的深层问题
- **品牌目标**：区别于纯娱乐型测试，突出"有趣又有深度"的产品调性

### 1.3 价值主张

| 价值维度 | 具体内容 |
|---------|---------|
| **用户价值** | 提供娱乐消遣 + 自我反思的双重体验，帮助用户识别职场中的不良行为模式 |
| **差异价值** | 区别于纯娱乐测试，通过"审判庭"形式引入哲学、社会学视角，增强内容深度 |
| **传播价值** | 搞笑测试结果 + 犀利点评天然具备社交传播属性 |

### 1.4 应用场景

- **使用平台**：微信小程序、H5网页
- **典型场景**：
  - 午休时间/通勤途中刷手机的用户
  - 职场社群内的分享传播
  - 朋友间的测试结果对比与讨论

### 1.5 项目背景

本项目是**职场相关产品比赛**的参赛作品，核心创新点在于：
1. 在现有"职场毁型人格测试"基础上增加"人物审判庭"模块
2. 参考李继刚老师的历史人物圆桌讨论Skill框架
3. 通过"审判"形式强化用户的批判性反思体验

## 2. 需求概述

### 2.1 用户痛点

| 痛点类型 | 具体描述 |
|---------|---------|
| **认知浅层化** | 市面现有的人格测试多为纯娱乐性质，用户测完即忘，缺乏深度反思 |
| **内容同质化** | 大量鸡汤式职场建议缺乏新意，无法引起用户共鸣 |
| **自省门槛高** | 普通人难以主动进行深度的自我行为分析，需要外部刺激引导 |

### 2.2 业务价值

- **内容价值**：通过"审判庭"形式创造独特的内容体验，形成差异化竞争
- **用户价值**：帮助用户在娱乐中完成对自身职场行为的批判性审视
- **传播价值**：测试结果 + 犀利点评具备强社交传播属性，可降低获客成本

### 2.3 预期收益

| 指标 | 预期目标 |
|-----|---------|
| 测试完成率 | ≥ 70% |
| 审判结果页分享率 | ≥ 25% |
| 人均阅读时长 | ≥ 90秒 |
| 用户满意度 | ≥ 4.0/5.0 |

### 2.4 版本范围

- **V1.0**：完整实现测试模块 + 8种人格的审判庭 + 结果分享功能
- **V2.0（规划）**：支持用户自定义审判人物、生成个性化反思清单

## 3. 用户需求与功能需求

### 3.1 用户需求

#### 3.1.1 目标用户

| 用户画像 | 描述 |
|---------|------|
| **职场新人** | 入职1-3年，对职场规则尚在适应期，渴望被认可又容易踩坑 |
| **职场老油条** | 工作5年以上，可能已形成某些固定行为模式，需要被"照镜子" |
| **摸鱼爱好者** | 以娱乐消遣为主，对深度内容需求不高 |
| **自我提升者** | 有较强的自我反思意愿，希望通过测试获得成长建议 |

#### 3.1.2 使用场景

**场景一：主动测试**
- 触发条件：用户在社交媒体看到分享/被朋友安利
- 用户行为：点击进入测试 → 完成人格测试题 → 获取测试结果 → 阅读审判庭
- 期望结果：获得有趣的测试结果 + 犀利的深度点评

**场景二：社交分享**
- 触发条件：用户完成测试，对结果满意或想与朋友比较
- 用户行为：点击分享按钮 → 选择分享渠道 → 分享给好友/朋友圈
- 期望结果：生成可分享的测试结果图片/链接

**场景三：围观审判**
- 触发条件：用户看到他人分享的测试结果
- 用户行为：点击链接 → 查看审判过程 → 自己也做测试
- 期望结果：被审判内容吸引，产生自己也测试的冲动

#### 3.1.3 核心需求

| 需求层级 | 具体描述 |
|---------|---------|
| **功能性需求** | 完成人格测试，获取趣味结果，了解审判过程 |
| **娱乐性需求** | 测试题目有趣，结果出人意料，审判点评犀利 |
| **社交性需求** | 测试结果可分享，话题性强，能引发讨论 |
| **成长性需求** | 获得有价值的反思建议，帮助改善职场表现 |

## 5. 功能范围

### 5.1 In Scope（包含功能）

#### 5.1.1 人格测试模块

| 功能 | 详细说明 |
|-----|---------|
| **测试题目** | 共10道选择题，每题4个选项，基于MBTI框架设计 |
| **人格类型** | 8种"毁型职场人格"：害群之马、甩锅宗师、人形摆烂机、汇报表演艺术家、御用背锅人、开会幽灵、永动打工人、人形复读机 |
| **人格判定** | 根据答题结果计算人格类型，输出类型名称 + tagline + 特征描述 |
| **结果动画** | 人格揭晓时的趣味动画效果 |

#### 5.1.2 人物审判模块

| 功能 | 详细说明 |
|-----|---------|
| **审判庭配置** | 每种人格配置3位历史/当代人物作为"法官" |
| **人物卡片** | 展示人物头像（AI生成或版权图片）、姓名、身份标签 |
| **观点内容** | 每位人物输出：开场白 + 核心观点 + 经典引用 + "简言之"一句话总结 |
| **观点展示** | 支持自动播放/手动滑动，自动播放间隔3秒 |
| **视觉风格** | 模拟"法庭审判"场景，有庄重感但不沉闷 |

**审判内容说明**：
- **内容数量**：8种人格 × 3位审判人物 = 24套审判内容
- **内容产出**：24套审判内容将另行输出专门文档，内容需符合李继刚老师"圆桌讨论"框架的设计原则
- **当前版本**：V1.0开发阶段使用测试内容跑通流程，待24套正式内容产出后替换

#### 5.1.3 结果输出模块

| 功能 | 详细说明 |
|-----|---------|
| **主持人综述** | 总结3位人物的共同观点，提炼核心争议点 |
| **反思建议** | 生成3-5条反思建议，帮助用户改善职场表现 |
| **分享卡片** | 生成可分享的图片（包含人格类型 + 一句话总结 + 二维码） |
| **分享渠道** | 支持微信好友、朋友圈、复制链接 |

#### 5.1.4 系统功能

| 功能 | 详细说明 |
|-----|---------|
| **进度保存** | 自动保存答题进度，支持中断后继续 |
| **历史记录** | 保存最近5次测试记录（本地存储） |
| **埋点采集** | 采集关键行为数据，用于产品优化 |

---

### 5.2 Out of Scope（不包含功能）

| 功能 | 说明 | 规划版本 |
|-----|------|---------|
| **自定义审判人物** | 用户不能选择其他人物参与审判 | V2.0 |
| **AI互动问答** | 用户不能与审判人物对话 | V2.0 |
| **深度报告下载** | 不支持生成PDF/Word报告 | V2.0 |
| **社交评论功能** | 不支持在结果页评论互动 | V2.0 |
| **排行榜功能** | 不支持好友测试结果排行 | V3.0 |
| **人格配对功能** | 不支持与好友人格配对 | V3.0 |

## 9. 风险与异常

### 9.1 潜在风险

| 风险类型 | 风险描述 | 风险等级 | 应对措施 |
|---------|---------|---------|---------|
| **内容风险** | 审判观点可能冒犯用户（涉及职场负面评价） | 中 | 增加免责声明，观点用历史人物口吻表达 |
| **内容风险** | AI生成内容出现事实性错误 | 中 | 建立内容审核机制，关键引用需人工确认 |
| **内容风险** | 历史人物引用不当引发争议 | 低 | 人物选择需符合普世价值观，避免争议人物 |
| **技术风险** | AI服务调用成本超出预算 | 高 | 设置每日调用配额，启用缓存策略 |
| **技术风险** | 高并发导致服务崩溃 | 中 | 实施限流降级预案，弹性扩容 |
| **数据风险** | 用户隐私数据泄露 | 低 | 不采集敏感个人信息，本地存储加密 |
| **业务风险** | 测试结果被用户用于职场人身攻击 | 中 | 增加正向引导，强调自省而非评判他人 |
| **合规风险** | 涉及历史人物的版权/肖像权问题 | 低 | 使用AI生成头像或公共领域图片 |

### 9.2 边界条件

#### 9.2.1 业务边界

| 边界类型 | 边界值 | 超出处理 |
|---------|-------|---------|
| 测试次数 | 同一用户每天最多测试3次 | 提示"今天已经测够了，明天再来吧" |
| 答题时间 | 单题作答时间上限30分钟 | 自动保存并提示超时 |
| 内容长度 | 审判观点单段不超过500字 | 截断并提供"展开"选项 |
| 分享频率 | 单用户每小时最多分享10次 | 提示"分享太频繁了" |

#### 9.2.2 技术边界

| 边界类型 | 边界值 | 说明 |
|---------|-------|------|
| 题目数量 | 10题/测试 | 不可更改 |
| 人格类型 | 8种 | 不可增加（V2.0） |
| 审判人物 | 3位/人格 | 不可更改 |
| 并发连接 | 1000/秒 | 峰值处理能力 |
| 存储容量 | 100MB/用户 | 本地存储上限 |

### 9.3 安全策略

| 安全维度 | 策略内容 |
|---------|---------|
| **传输安全** | 全站HTTPS，API接口鉴权 |
| **数据安全** | 用户答题数据不包含个人信息 |
| **内容安全** | AI生成内容经过敏感词过滤 |
| **接口安全** | 限制单IP请求频率，防刷接口 |
